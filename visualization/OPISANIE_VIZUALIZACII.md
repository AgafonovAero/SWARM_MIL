# Описание визуализации

## Назначение визуализации

Слой визуализации предназначен для подготовки наблюдаемого демонстратора
моделирования роя БВС по уже рассчитанным данным этапов 2, 3, 4, 5 и 6.

Визуализация не изменяет результаты расчета, а только преобразует их в
форму, удобную для просмотра, анимации, графиков и краткого отчета опыта.

## Состав отображаемых данных

Слой визуализации использует:

- сценарий опыта;
- временную ось опыта;
- положения и скорости БВС;
- граф связности по времени;
- звенья роя и головные БВС;
- события передачи сообщений;
- итоговые показатели качества.

## Трехмерная сцена роя

Трехмерная сцена показывает область полета, текущие положения БВС,
траектории движения, линии связи, головные БВС и текстовое пояснение по
текущему моменту времени.

Сцена создается как в обычном видимом режиме для демонстратора, так и в
невидимом режиме для пакетной проверки этапа 7.

## Отображение БВС

Каждый БВС показывается как отдельная точка в трехмерном пространстве.
Цвет точки определяется принадлежностью БВС к звену текущего момента
времени. Подписи БВС используются для привязки к идентификаторам сценария.

## Отображение траекторий

Траектории строятся по уже рассчитанным положениям пассивной кинематики.
На этапе 7 траектории только отображаются и не изменяются средствами
визуализации.

## Отображение линий связи

Для каждого кадра могут отображаться линии связи, соответствующие матрице
смежности текущего графа связности. Линии связи показываются только для
наблюдения расчетного состояния роя.

## Отображение звеньев

Принадлежность БВС к звену показывается цветом участника. Для текущего
кадра сохраняются состав звеньев и их головные БВС.

## Отображение головных БВС

Головные БВС выделяются отдельным маркером повышенного размера. Такое
выделение является только средством отображения результата этапа 5.

## Отображение событий передачи сообщений

Для каждого кадра собираются события передачи сообщений, относящиеся к
текущему моменту времени. В сцене показывается краткое текстовое
пояснение по событиям передачи.

## Состав графиков показателей

Графики этапа 7 показывают:

- число линий связи во времени;
- среднюю степень БВС во времени;
- число звеньев во времени;
- число одиночных звеньев во времени;
- долю доставленных сообщений;
- накопленное число доставленных сообщений;
- накопленное число потерянных сообщений;
- среднюю задержку доставки;
- среднее число пересылок.

## Порядок запуска демонстратора

Для запуска демонстратора используется файл `zapusk_demonstratora_roya.m`
в корне проекта. Демонстратор по умолчанию работает со сценарием
`stroi_malyi`, подготавливает данные, открывает трехмерную сцену и
графики показателей.

## Ограничения этапа 7

- визуализация работает только по уже рассчитанным данным этапов 2–6;
- сцена не реализует управление движением;
- сцена не изменяет траектории БВС;
- в слое визуализации не добавляются новые способы маршрутизации;
- визуализация не реализует обучение;
- визуализация не реализует распределение ресурсов;
- модели Simulink на этапе 7 не создаются.

## Границы применимости

Демонстрационный опыт этапа 7 является исследовательским средством
наблюдения расчетных данных проекта. Он не является системой управления
реальными БВС, не предназначен для вредного применения и не содержит
практических инструкций по реальному использованию аппаратуры.
